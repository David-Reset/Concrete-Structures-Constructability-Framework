IF
Framework Image Formatter
Rename, resize and re-encode framework images — one or many at once, entirely in your browser
What this does
Load the current data.json, drop in one or several images, give each a plain name like pier hole depth, and download clean, resized files like pier-hole-depth.png for the GitHub images folder. Each name is checked for uniqueness against images already in the framework and the others in this batch. Re-encoding through the browser also strips original photo metadata (GPS, camera, device) — a built-in de-identification step.

3 · Output settings (applied to every image)

For the web, WebP is usually best — smallest files at the same quality, supported by every current browser. Use PNG for screenshots, diagrams or anything needing transparency; JPEG for maximum compatibility with old software; Keep for a mixed batch. Any row can override this below.
Shrinks only if longer; never enlarges.
Lower = smaller files. PNG ignores this.
Every downloaded file is re-encoded, which removes the original metadata (GPS coordinates, camera and device info). The forms ask submitters for de-identified images — this enforces it.

4 · Images to download

Name each image — a human-written name beats anything auto-generated. The filename is the lowercase, hyphenated version plus the chosen extension, checked for uniqueness as you type. Stuck on a unique name? Use the per-row Make name unique, or the button above to resolve every clash at once (it appends -2, -3, … to duplicates).
